# ADR-011 : Authentification et Layout — Sprint 1

**Statut** : Accepté  
**Date** : 2026-07-22  
**Approbation** : Équipe

## Contexte
Le Sprint 1 met en place l'authentification, le layout global et le routage.

## Décisions

### Auth
- **SignIn** : `supabase.auth.signInWithPassword()` direct, sans Edge Function Netlify. L'Edge Function était redondante pour une app mono-projet.
- **Rôle admin** : stocké dans `user.user_metadata.role`, lu au moment de la connexion. Permet d'éviter une requête supplémentaire à la table `profiles`.
- **Initialisation** : `getSession()` au mount, `onAuthStateChange` pour les mutations suivantes. `useRef` empêche la double exécution en dev (StrictMode React 18).

### Layout
- **Navbar** : sticky + scroll detection (> 50px). Mobile responsive avec toggle menu.
- **Footer** : liens du club + réseaux sociaux.
- **Providers** : `ThemeProvider` → `AuthProvider` (ordre : thème avant auth pour les styles).

### Routing
- Lazy loading pour toutes les pages (code splitting).
- `PrivateRoute` : redirection vers `/login` si non connecté, vers `/` si non admin.
- `*/register` → redirige vers `/login`.

## Conséquences
- Pas de dépendance serveur Edge Function en local.
- Risque : si le rôle est stocké en BDD et pas dans `user_metadata`, l'admin ne sera pas reconnu. Mitigation : synchroniser via trigger Supabase ou vérifier aussi dans `profiles`.
- Le code splitting améliore les perfs mais ajoute un léger délai au chargement initial de chaque page.
