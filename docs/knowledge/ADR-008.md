# ADR-008 : Notifications push

**Status** : Accepted  
**Date** : 2026-07-23  
**Context** : Les supporters doivent être alertés en temps réel des prochains matchs, résultats et actualités. L'objectif est d'augmenter l'engagement et le taux de rétention.
**Decision** : Utiliser l'API Push Notification native du navigateur via le Service Worker de la PWA existante. Les abonnements sont stockés dans Supabase (table `push_subscriptions`). L'envoi des notifications est déclenché depuis l'interface admin via une Edge Function Netlify utilisant `web-push`.
**Consequences** : 
  - Intégration native navigateur, pas de dépendance externe pour l'affichage
  - Stockage centralisé des abonnements dans Supabase
  - Edge Function Netlify pour l'envoi (coût négligeable)
  - L'utilisateur doit donner son autorisation (permission prompt)
  - Maintenance : les clés VAPID doivent être renouvelées périodiquement
**Conformité** : RGPD — consentement explicite requis pour l'abonnement
**Approval** : Équipe, 2026-07-23
