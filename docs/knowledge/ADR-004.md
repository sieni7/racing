# ADR-004 : Design System

**Statut** : Accepté  
**Date** : 2026-07-22  
**Approbation** : Équipe

## Contexte
Le projet a besoin d'une identité visuelle cohérente et de composants réutilisables pour toutes les pages publiques.

## Décisions

### Palette de couleurs
- **Primaire** : `#F97316` (orange Racing Club) — boutons, accents, titres
- **CTA** : `#EA580C` (orange foncé) — hover des boutons, liens
- **Bleu club** : `#1E3A5F`, `#1A3C6E` — fonds hero, header/footer
- Extensions orange (50-900) et bleu (50-900) pour les variations

### Typographie
- **Titres (display)** : `Space Grotesk` — grappes h1-h3, hero, grandes sections
- **Corps (sans)** : `Inter` — textes courants, cartes, navigation

### Composants UI
- Tous les composants atomiques dans `src/components/ui/`
- Chaque composant accepte son entité typée via une prop unique (`player`, `match`, `news`, `member`)
- Effets hover via `group-hover:` Tailwind (scale, shadow)
- Skelettes de chargement : `Skeleton`, `CardSkeleton`, `ListSkeleton`

### Framework CSS
- Tailwind CSS 3 avec `darkMode: 'class'`
- Variantes dark systématiques sur tous les composants

## Conséquences
- Code modulaire, design cohérent entre toutes les pages
- Facilité de maintenance : modifier un composant affecte toutes ses occurrences
- Dark mode maintenu sans effort grâce aux variantes Tailwind
- Pas de dépendance à une librairie de composants externe (headless UI, shadcn)
